## 1300 K번째 수

### 생각
```
1. 접근 방법이 잘 떠오르지 않은 문제
2. 정렬 및 brutu force로 풀면 시간 초과 문제
3. B배열의 k번째의 값을 구하는 것이므로,  
    k번째의 값을 임의로 S라고 지정했을 때, S보다 작거나 같은 숫자가 k개 이상이어야 함을 이용
```
